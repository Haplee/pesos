# Sistema de instrucciones — Fran (ASIR)

## Objetivo

Quiero que reescribas todo el código del proyecto manteniendo exactamente la misma funcionalidad, pero adaptándolo a mi estilo personal como desarrollador.

---

## Estilo de código

- Programo en español: el código puede estar en español o, si procede, con comentarios en español.
- Prioridad absoluta a la legibilidad.
- Uso nombres descriptivos y consistentes en inglés.
- Evito sobreingeniería y patrones innecesarios.
- Prefiero código claro, directo, práctico y fácil de mantener.
- La estructura del proyecto debe ser lógica y ordenada.
- Los comentarios solo deben usarse cuando aporten valor real, no para explicar lo obvio.

---

## Stack del proyecto

- React + TypeScript
- Tailwind CSS
- Zustand
- Vite
- Tests y Docker solo si ya existen en el proyecto

---

## Reglas estrictas

### No hacer

- No cambiar la lógica ni el comportamiento.
- No eliminar funcionalidades.
- No cambiar el stack.
- No añadir dependencias innecesarias.
- No introducir arquitecturas complejas si una solución simple basta.

### Sí hacer

- Refactorizar para mejorar claridad y coherencia.
- Unificar estilo en todo el proyecto.
- Extraer helpers o utilidades solo si mejora la legibilidad.
- Mejorar la organización de carpetas si tiene sentido.
- Actualizar rutas y referencias si se mueve algún archivo.
- Eliminar archivos residuales, temporales o basura al finalizar.

---

## Resultado esperado

- Código reescrito completo.
- Estilo consistente en todos los archivos.
- El proyecto debe parecer escrito por una sola persona.
- Siempre asumir la solución más simple y práctica.
- Si se modifica algo existente, conservar su funcionalidad exacta.
